# 接妆管理助手 - 产品需求文档 (PRD)

## 文档信息

| 项目 | 内容 |
|------|------|
| 产品名称 | 接妆管理助手 |
| 版本 | v1.0 |
| 日期 | 2026-07-08 |
| 目标用户 | 自由职业化妆师及其客户 |
| 平台 | 移动端优先的响应式Web应用 |

---

## 1. 产品概述

### 1.1 背景

自由职业化妆师在日常接单过程中，面临以下痛点：
- 妆价信息反复被客户询问，沟通成本高
- 预约时间全靠微信沟通，容易撞单、漏单
- 缺少对收入的系统化统计，财务管理混乱
- 客户需求（角色参考图、偏好等）容易遗漏

### 1.2 产品定位

一款面向个人化妆师的轻量级接妆管理工具。化妆师将链接分享给客户，客户打开即可查看妆价、自助预约，化妆师端可管理全部业务流程并查看收入统计。

### 1.3 核心价值

| 角色 | 价值 |
|------|------|
| 化妆师 | 减少重复沟通、避免撞单、清晰掌握排期和收入 |
| 客户 | 自助查价、自助预约、体验专业化服务 |

---

## 2. 用户角色

### 2.1 管理员（化妆师本人）

- 系统唯一管理员，拥有全部权限
- 可编辑注意事项、妆价、可约时间
- 可查看所有预约详情及收入统计
- 可审核/确认/取消预约

### 2.2 普通用户（客户）

- 通过注册/登录进入系统
- 只读查看注意事项和妆价
- 可在可用时段内进行预约
- 可查看和管理自己的预约记录

## 3. 功能模块详细设计

### 3.1 注册与登录

#### 3.1.1 注册

| 字段 | 必填 | 说明 |
|------|------|------|
| 手机号 | 是 | 作为登录账号，唯一标识 |
| 昵称 | 是 | 用于预约记录中显示 |
| 密码 | 是 | 6-20位，须包含数字和字母 |

- 管理员账号预先内置，不通过注册流程创建
- 所有新注册用户默认为「客户」角色

#### 3.1.2 登录

- 手机号 + 密码登录
- 支持「记住我」保持登录状态（7天有效）
- 登录后根据角色跳转至对应视图

### 3.2 注意事项弹窗

#### 触发时机
- 每次登录后、进入主页面前弹出

#### 化妆师端
- 可编辑注意事项内容（富文本，支持加粗、换行、列表）
- 实时预览编辑效果
- 保存后立即对所有客户生效

#### 客户端
- 以弹窗形式展示注意事项全文
- 底部显示「我已知晓」按钮
- 点击后方可进入系统主页面
- 未点击前不可关闭弹窗或进入系统

#### 典型内容示例
> - 预约后如需取消，请提前48小时告知
> - 请自备隐形眼镜、美瞳等个人用品
> - 上门服务需提前确认地址，超出10公里加收交通费
> - cos妆请提前发角色参考图

### 3.3 页面一：妆价列表

#### 3.3.1 化妆师端 - 妆价管理

**妆品项目管理：**

| 字段 | 必填 | 说明 |
|------|------|------|
| 妆型名称 | 是 | 如：新娘妆、cos妆、日常妆、写真妆 |
| 价格（元） | 是 | 支持整数和一位小数 |
| 描述 | 否 | 对该妆型的简要说明，如包含什么服务 |
| 排序权重 | 否 | 控制展示顺序，默认按添加时间排列 |

**附加服务管理：**

| 字段 | 必填 | 说明 |
|------|------|------|
| 服务名称 | 是 | 如：上门服务、早妆加急、补妆跟妆 |
| 加收金额（元） | 是 | 在妆价基础上额外加收的费用 |
| 描述 | 否 | 补充说明，如适用范围 |

**操作：**
- 新增 / 编辑 / 删除妆品项目和附加服务
- 可设置某项为「暂不接单」状态（灰显但不删除）
- 拖拽排序

#### 3.3.2 客户端 - 妆价查看

- 卡片式布局展示所有妆型及价格
- 附加服务单独区域展示
- 「暂不接单」的项目灰显并标注状态
- 底部可选的「立即预约」快捷入口，跳转至预约页面

### 3.4 页面二：预约管理

#### 3.4.1 化妆师端 - 排期管理

**设置可接妆日期：**
- 日历视图，可逐日或批量设置「可约」/「不可约」
- 对每个可约日期，设置可用时间段（如 06:00-20:00，以30分钟为粒度）
- 支持设置每个时间段最多接待人数（默认1人）
- 可一键复制某日设置到其他日期

**预约列表：**
- 以日历或列表形式查看所有预约
- 每条预约展示：客户昵称、手机号、预约时间、妆型、备注、参考图片
- 操作：确认预约 / 取消预约（需填写取消原因）
- 预约状态流转：待确认 → 已确认 → 已完成 / 已取消

**时间表生成：**
- 按日 / 按周 / 按月查看排期
- 时间轴视图，直观展示每日的预约分布
- 支持导出为图片（方便分享到朋友圈或微信群）

#### 3.4.2 客户端 - 预约流程

**第一步：选择日期**
- 日历视图展示
- 可约日期高亮显示，不可约日期置灰
- 已约满的日期标注「已约满」

**第二步：选择时间段**
- 展示所选日期下的可用时间段
- 已被其他客户预约的时间段标注「已约」，不可选
- 剩余可约时间段可正常选择

**第三步：填写预约信息**

| 字段 | 必填 | 说明 |
|------|------|------|
| 妆型选择 | 是 | 从妆价列表中选择 |
| 附加服务 | 否 | 可多选 |
| 备注 | 否 | 自由文本，如：偏好风格、角色名称、特殊要求 |
| 参考图片 | 否 | 支持上传最多6张图片（每张不超过5MB） |
| 联系方式 | 是 | 自动填入注册手机号，可修改或补充微信号 |

**第四步：确认提交**
- 展示预约汇总（日期、时间、妆型、价格明细、总价）
- 确认无误后提交

**我的预约：**
- 查看自己的历史预约记录
- 可取消「待确认」状态的预约
- 查看每条预约的当前状态

### 3.5 页面三：收入统计（仅化妆师可见）

#### 3.5.1 月度概览

| 指标 | 说明 |
|------|------|
| 本月预期收入 | 所有「已确认」预约的总金额 |
| 本月已完成收入 | 所有「已完成」预约的总金额 |
| 本月预约数 | 已确认 + 已完成的预约总数 |
| 本月取消数 | 被取消的预约数 |

#### 3.5.2 收入明细

- 按日期排列的收入明细列表
- 每条记录展示：日期、客户昵称、妆型、附加服务、总金额、状态
- 支持按月切换查看历史数据

#### 3.5.3 数据统计

- 各妆型接单占比（饼图）
- 月度收入趋势（折线图，展示近6个月）
- 热门预约时段分布（柱状图，便于化妆师优化排期）

#### 3.5.4 手动记账

- 支持手动添加线下收入记录（非通过系统预约的单子）
- 字段：日期、客户名称（选填）、妆型、金额、备注
- 手动记录与系统预约记录共同纳入统计

## 4. 补充功能（产品经理建议新增）

### 4.1 作品集展示

- 化妆师可上传作品照片，分类展示（新娘妆、cos妆等）
- 客户在查看妆价时可查看对应妆型的作品案例
- 帮助客户做出预约决策，也是化妆师的对外展示窗口

### 4.2 消息通知

- 新预约提醒（化妆师端）
- 预约状态变更通知（客户端）
- 预约前一天自动提醒（双方）
- 通知方式：系统内通知 + 浏览器推送（如用户授权）

### 4.3 客户评价（可选，v2考虑）

- 服务完成后客户可对本次服务评分（1-5星）+ 文字评价
- 化妆师可选择是否公开展示评价
- 好评可增强新客户信任

### 4.4 分享获客

- 生成个人专属页面链接 / 二维码
- 客户可通过链接直接访问化妆师的妆价页面
- 方便化妆师在社交媒体、朋友圈推广

## 6. 非功能性需求

### 6.1 性能

- 首屏加载时间 < 2秒（4G网络环境）
- 图片上传支持前端压缩，减少流量消耗
- 日历组件交互流畅，无卡顿

### 6.2 兼容性

- 移动端优先，适配 iPhone SE ~ iPhone Pro Max 及主流安卓机型
- 支持微信内置浏览器打开（核心使用场景）
- 桌面端浏览器可用但非重点优化目标

### 6.3 安全

- 密码加密存储（bcrypt）
- 客户间数据隔离，客户只能看到自己的预约
- 收入页面严格限制为管理员角色访问
- 图片上传限制类型（jpg/png/webp）和大小

### 6.4 数据

- 预约记录永久保留
- 图片存储需考虑存储成本（建议压缩后存储）
- 支持数据导出（CSV格式，化妆师可下载收入记录）

## 7. 技术方案建议

| 层级 | 建议方案 | 理由 |
|------|----------|------|
| 前端 | React + Tailwind CSS + Mobile-first | 组件生态丰富，移动端适配友好 |
| 后端 | Node.js (Express/Koa) | 轻量高效，适合个人项目 |
| 数据库 | SQLite / PostgreSQL | SQLite适合初期，PostgreSQL便于后续扩展 |
| 图片存储 | 本地文件系统 / 云存储(OSS) | 初期本地即可，量大后迁移云存储 |
| 部署 | Vercel / 轻量云服务器 | Vercel零配置部署，适合前后端一体项目 |

## 8. 页面原型要点

### 8.1 视觉风格

- 整体风格：简约、温柔、专业感
- 主色调：建议使用温暖的玫瑰金色系或裸粉色系，贴合美妆行业调性
- 字体：清晰易读，中文使用系统默认字体
- 图标：线性图标风格，轻量不花哨

### 8.2 关键交互

- 底部Tab导航栏（移动端）
- 日历组件：滑动切换月份，点击选择日期
- 妆价卡片：圆角卡片布局，价格醒目
- 预约流程：分步引导，进度指示器
- 弹窗：注意事项使用全屏弹窗，操作确认使用底部半屏弹窗

## 10. 风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| 客户不习惯在线预约 | 使用率低 | 保留手动记账功能，线上线下并行过渡 |
| 微信内置浏览器兼容问题 | 功能异常 | 重点在微信浏览器中测试，处理已知兼容问题 |
| 图片存储成本增长 | 运维成本上升 | 前端压缩 + 限制上传数量和大小 |
| 单点管理员 | 化妆师无法登录时系统瘫痪 | 提供密码找回机制 |

## 附录：名词解释

| 术语 | 说明 |
|------|------|
| 妆型 | 化妆服务的类型，如新娘妆、cos妆、日常妆等 |
